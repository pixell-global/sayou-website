The Problem

The web era built tools for transactions. Not for thinking.

Databases were designed for CRUD.

Create, read, update, delete. Every record is a row, every relationship a join, every nuance stripped to fit a schema. A CRM stores a name and a dollar amount — not the reasoning behind the deal. Structure without context.

Agents think deeply — then forget everything.

Your agent can research, analyze, and reason. But when the session ends, the thinking disappears. Tomorrow it starts from zero. There's nowhere for understanding to accumulate.

사유 [sayu]
to contemplate deeply and reason broadly

In Korean philosophy, 사유 is the act of sustained, layered thinking — where each insight builds on the last. It's the opposite of forgetting.

We built sayou to give AI agents the same ability: a place where reasoning persists, context accumulates, and understanding compounds over time.

How It Works

Files that hold what databases can't.

Structure and context in one file

YAML frontmatter for structured fields. Markdown body for reasoning, notes, and context. One file holds what a database splits across tables — queryable, human-readable, and natively understood by LLMs.

Knowledge that compounds

Every write creates a new version. Nothing is lost. The full history of how understanding evolved — visible, auditable, reversible.

Any agent can connect

MCP server for Claude Code, Cursor, and any MCP-compatible tool. Python library for direct integration. No vendor lock-in. Bring your own agent framework.

The Unit

One read. Full context.

clients/acme-corp.md
---
company: Acme Corp
status: active
stage: proposal_sent
deal_size: $180,000
last_contact: 2025-02-12
---

# Acme Corp

## Context
- Evaluated three competitors before us
- Technical team is skeptical, needs POC
- Budget approved through Q3

## Meeting Notes
- Feb 10: Reviewed draft, wants deeper analysis
- Jan 20: Kickoff, aligned on scope

In a traditional CRM, your agent needs:

  • A contacts query for the name and company
  • A deals query for stage and value
  • An activities query for meeting notes
  • A notes query for the unstructured context

In sayou, the agent reads one file and has everything.

Frontmatter for structure. Markdown for context. Versioned. Auditable. Readable by any agent — or any human.

In Practice

How agents use sayou

Two interfaces, one workspace. Use the Python SDK in your agent code, or connect as an MCP server for Claude Code and Cursor.

Python SDK — Day 1
from sayou import Workspace

async with Workspace() as ws:
    # Agent stores today's competitive analysis
    await ws.write(
        "research/market/2025-02-14.md",
        """---
status: draft
topic: competitive-landscape
---
# Market Analysis

Competitor X launched a new product targeting
mid-market. Pricing undercuts ours by 20%..."""
    )

Not a vector database. Stores files, not embeddings.

Not a memory layer. Stores work product, not conversation snippets.

Not a sandbox. Persistent storage, not ephemeral execution.

Not a filesystem. A knowledge workspace with versioning and indexing.

Open Source

Built in the open

Apache 2.0 from day one. Infrastructure should be a shared standard, not a proprietary moat. The infrastructure layers that win are the ones that become open standards. sayou aims to be the standard for agent persistent storage.
